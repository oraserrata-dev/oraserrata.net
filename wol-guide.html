Enable Wake-on-LAN on Windows

Five minutes, once, on the PC you want Rouse to wake.

Wake-on-LAN lets a sleeping PC be woken by a "magic packet" sent over your network. Windows PCs usually ship with it half-enabled: the hardware supports it, but two or three settings stand in the way. Work through the steps below on the target PC, then use Test the Wake in Rouse's Setup Guide to confirm.

⚡
Use Ethernet. Wake-on-LAN over Wi-Fi is unreliable on most hardware. If the PC has an Ethernet port, plug it in. These steps assume the wired adapter.

1Allow the network adapter to wake the PC

Right-click the Start button and choose Device Manager. Expand Network adapters, double-click your Ethernet adapter, and open the Power Management tab. Check the first two boxes:

The second box is optional but recommended: it stops ordinary network chatter from waking the PC at random.

2Enable Wake on Magic Packet

In the same Properties window, open the Advanced tab. In the Property list, find Wake on Magic Packet and set its Value to Enabled. Names vary slightly by adapter. Realtek calls it "Wake on Magic Packet"; some list "WOL & Shutdown Link Speed" too, which can stay at its default.

3Turn off Fast Startup

This is the one most guides miss. With Fast Startup on, a "shut down" Windows PC parks in a hybrid state where many adapters stop listening for magic packets, so wake works from sleep but not from shutdown.

Open Control Panel → Power Options → Choose what the power buttons do, click Change settings that are currently unavailable, and untick Turn on fast startup:

Despite the "(recommended)" label, the boot-time difference on a modern SSD is a second or two, and reliable remote wake is worth far more.

4Check the BIOS / UEFI

Most PCs ship with this on, so try a test wake first. If wake from full shutdown still fails, reboot into BIOS/UEFI setup (usually Del or F2 at power-on) and look for a setting named one of:

  • Wake on LAN / WOL
  • Power On by PCI-E / PCIe device
  • Resume by LAN, or ErP / EuP mode (set ErP to Disabled, since ErP cuts standby power to the adapter)

It usually lives under Power Management, Advanced, or APM settings. Enable it, save, and exit.

5Test it with Rouse

Put the PC to sleep, then tap its wake button in Rouse, or run Test the Wake in the Setup Guide, which sends a magic packet and tells you what it did. If sleep-wake works but shutdown-wake doesn't, revisit steps 3 and 4.

Waking a Mac instead?

Macs are simpler: turn on Wake for network access in System Settings → Battery (desktops: Energy). Note that Apple Silicon Macs return to a light sleep quickly after waking; install Rouse on the target Mac and enable Wake Helper to hold it awake for remote-desktop connections. See Support for details.

Still stuck?

Wake-on-LAN behavior varies a lot by adapter and motherboard. If these steps don't work, search your PC or motherboard model plus "Wake on LAN" for vendor-specific settings, and check that your Ethernet driver is current.
